# VirtualWorks Lab Internship Projects

This repository contains a collection of full-stack web applications developed as part of the **VirtualWorks Lab** virtual internship program. The program is designed to provide project-based, real-world work experience focusing on building production-level features and engaging in professional development.

## 🛠️ Global Technology Stack
Across all projects in this repository, a consistent and robust technology stack is utilized:
- **Frontend:** HTML5, CSS3 (Vanilla), JavaScript (Vanilla, Fetch API)
- **Backend:** Python, Flask (RESTful APIs)
- **Database:** SQLite3
- **Design:** Modern UI/UX principles, including dark mode, glassmorphism, responsive layouts, and micro-animations.

## 🚀 Completed Internship Projects

Below is the list of the 5 completed tasks, each contained within its own dedicated directory.

### 1. [Coffee Rating App](./coffee_rating_app/)
A modern, responsive web application for rating different types of coffee. It features asynchronous voting, dynamically updating vote counts without page reloads, and persistent storage of user interactions.

### 2. [Profile Card Generator](./profile_card_generator/)
A web application designed to collect user information via a form and dynamically generate a modern, responsive, and visually appealing profile card template using the provided data.

### 3. [Quick Note](./quick_note/)
A single-page application (SPA) designed for rapid note-taking. It implements a fully asynchronous RESTful API to perform CRUD (Create, Read, Update, Delete) operations on notes directly stored in the SQLite database, ensuring seamless transitions without page reloads.

### 4. [Quote Generator with History](./quote_generator/)
An application that fetches random quotes from a public external API, elegantly displays them to the user, and automatically logs each generated quote into a persistent history list within the database.

### 5. [Team Availability Tracker](./team_availability_tracker/)
A clean dashboard application that displays a list of team members and provides toggle buttons to update their availability status (Available/Busy) in real-time. It relies on conditional rendering and state synchronization with the backend database.
